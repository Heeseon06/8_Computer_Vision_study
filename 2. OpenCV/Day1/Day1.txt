영상(Image)
- 픽셀이 바둑판 모양의 격자에 나열되어 있는 형태
- 픽셀: 이미지를 구성하는 가장 작은 단위
- 2차원 행렬

그레이스케일 영상
- 흑백 사진처럼 색상 정보가 없는 영상
- 밝기 정보만으로 구성된 영상
- 밝기 정보는 256단계로 표현(0 ~ 255)
- 8bit에 저장 = 1byte = numpy.uint8
- 가로크기 * 세로크기 = 이미지 용량
    예) 28 * 28 = 784 bytes

트로컬러 영상
- 컬러 사진처럼 색상 정보를 가지고 있기 때문에 다양한 색상을 표현할 수 있는 영상
- red, green, blue 색 성분을 사용하고 각 256단계로 표현
- 픽셀의 표현 -> (255, 255, 255) 튜플 형태로 표현
- 3byte = numpy.ndarray
- 가로크기 * 세로크기 * 3 = 이미지 용량
    예) 28 * 28 * 3 = 2352 bytes

영상 파일 형식
BMP
_ 픽셀 데이터를 압축하지 않고 그대로 저장
- 용량이 매우 큼(픽셀이 일반적인 크기)
- 파일 구조가 단순해서 별도의 라이브러리 없이 프로그래밍이 가능
JPG(JPEG)
- 압축률이 좋아서 파일 용량이 크게 감소
- 사진과 같은 컬러영상을 저장
- 손실 압축
GIF
- 움직이는 영상 지원
- 256색 이하의 영상을 저장
- 무손실 압축
PNG
- 웹 이미지용으로 권장
- 무손실, 손실 압축
-알파 태널(투명도)을 지원
- (255, 255, 255, 1) => 차원이 다름

OpenCV-Python 모듈

opencv
- 컴퓨터 비전와 이미지 처리를 위한 오픈소스 라이브러리
    - 1999년 인텔에서 영상처리 기술을 개발하기 위한 목적으로 만듦
    - 2000년 BSD 라이센스 배포
    - 2011년 이후 OpenCV2로 개발 시작
- OpenCV 설치
    - pip install opencv-python => 설치
    - import cv2 => import


동영상 출력하기
FourCC(Four Character Code)
- 4바이트로 된 문자열이며 데이터 형식을 구분하는 고유 글자
- 주로 AVI 파일의 영상 코덱을 구분할 때 사용
    cv2.VideoWriter.fourcc(*'DIVX') // Divx
    cv2.VideoWriter.fourcc(*'XVID') // Xvid
    cv2.VideoWriter.fourcc(*'MP4V') // mp4
    ...
